Publishing Changes

The Publishing Tag Set, journalpublishing3.dtd, was brought up to version 3.0, with the DTD version attribute (@dtd-version) set to “3.0 ” and the version and date changed in every module to “v3.0 20080202 ”.

Publishing Changes in Classes

The following changes were made to journalpubcustom-classes3.ent:

  • Appearance Class / Font — Deleted the element fontfrom the Tag Sets, which removed it from %appearance.class;. Then removed %appearance.class; altogether, as <hr> in the Publishing Tag Set was only allowed inside table cells, and those horizontal rules should now be accomplished using the XHTML attributes.
  • References Class — The Publishing Tag Set no longer overrides %references.class;, which is used in the models of <mixed-citation>, <element-citation>, <product>, <related-article>, and <related-object>.
    In journalpubcustom-classes3.ent, removed the %references.class; as duplicative of the Suite default.
    In default-classes3.ent, in %references.class;, added the elements:
    • <chapter-title>,
    • <part-title>,
    • <date-in-citation>,
    • <size>,
    • <string-name>, and
    • <volume-series>.

    Also deleted the following elements from %references.class;:
    • <access-date> (replaced by <date-in-citation>),
    • <page-count> (replaced by <size>), and
    • <time-stamp> (replaced by <date-in-citation>).

    The current default references class (used by all Tag Sets) is now:

  • Roman Emphasis — Added <roman> to the %emphasis.class;.

Publishing Journal Metadata

In journalpubcustom-models3.ent, in the model for <journal-meta>:

  • ISBN — Added <isbn>, optional and repeatable following <issn>.
  • Journal Title Group — Removed the following sequence: 
    journal-title*, journal-subtitle*,
trans-title*, trans-subtitle*, 
abbrev-journal-title*,

    and replaced it with the new element <journal-title-group> (optional, repeatable) inside <journal-meta> (using %journal-meta-model;): 
    (journal-id*, journal-title-group*, issn*, publisher?, 
notes?, custom-meta-group?)

    Inside the model for <journal-title-group>, replaced the sequence: 
    trans-title, trans-subtitle*

    with the new element <trans-title-group>, optional repeatable.
  • Translated Title Group — In common3.ent, created a new element <trans-title-group> with the model (trans-title, trans-subtitle*), and gave <trans-title-group> all the attributes that <trans-title> had (@content-type, @id (of type ID), and @xml:lang). Made the @xml:lang required on the <trans-title-group>, leaving it as optional on <trans-title>, since this element is now used only be used inside a <trans-title-group> or inside citations. Because of the usage inside citations, the attributes were left on <trans-title>, although, for an such a title inside a <trans-title-group>, the attributes should be on the outer container element.
  • Thus the <journal-meta> override is: 
    (journal-id+, journal-title-group*, 
issn+, isbn*, publisher?, notes?)

Other changes to article metadata elements:

  • Author Notes — In journalpubcustom-models3.ent, removed the parameter entity %author-notes-model;, since Publishing now uses the default model, which is defined in articlemeta3.ent.

Publishing Expand model of <front-stub>

In journalpublishing3.dtd, enhanced the <front-stub> model to include all elements already present in <article-meta>. Therefore, in the Publishing Tag Set, the model for <front-stub> now differs from <article-meta> only by allowing the grouping element <title-group> (which includes the article title and translated titles) to be optional, and the <pub-date> to be optional and repeatable. This model should accommodate any publisher’s subset of metadata for a <sub-article> while retaining the convenience of not requiring the repetition of the full infrastructure that must be present inside <front> for the main article.

Publishing Chemical Structures Changes

Although it was not the intent when the Tag Sets were written, most Tag Set users have been treating <chem-struct> as an inline and <chem-struct-wrap> as a block structure. This greatly simplifies all processing with the two and seems quite flexible, so steps were taken to regularize this usage:

  • As an inline object, <chem-struct> should be able to be used anywhere that <inline-formula> is used; therefore, <chem-struct> has been added to %inline-math.class; (in default-classes3.ent).
  • As an inline object, <chem-struct> should not be used as a block-level object, so it was removed from %block-display.class; (in default-classes3.ent).
  • As an inline object, <chem-struct> must be inside <chem-struct-wrap> (using %chem-struct-wrap-model;) (in display3.ent).
  • As a block object, <chem-struct-wrap> needs to be everywhere block display objects are used, so it needs to be part of the %block-display.class; and %floats-display.class;, which it already was in display3.ent.
  • As a block object, <chem-struct-wrap> should be inside figures (using %fig-model;), which it already was in the Publishing override.
  • Changed the OR group allowed inside a <table-wrap> (using %inside-table-wrap;) to allow <chem-struct-wrap> instead of <chem-struct> (in display3.ent).

The element <chem-struct-wrap> has been removed from within the following elements, where it has been allowed in previous versions, allowing only the inline <chem-struct> to be used. [Backward incompatible]

  • <td> and <th> (using %inside-cell;)
  • <term> (using %term-elements;)

The element <chem-struct> has been removed from within the following elements, where it has been allowed in previous versions, allowing only the block <chem-struct-wrap> to be used. [Backward incompatible]

Publishing Back Matter

Publishing Reference Lists

The Publishing Tag Set no longer has an override for the %references.class;. This Tag Set uses the default references elements.

  • New Elements in Citations — In default-classes3.ent, added the following new elements to %references.class;:
    • <chapter-title>,
    • <date-in-citation> (replaces <time-stamp> and <access-date>),
    • <issue-part>,
    • <part-title>,
    • <series> (a requested addition; was already present in the class),
    • <size> (replaces <page-count>), and
    • <volume-series>.
  • Date Elements Deleted from Citations — In default-classes3.ent, deleted the following elements from %references.class;: [Backward incompatible]
    • <access-date> (replaced by <date-in-citation>)
    • <time-stamp> (replaced by <date-in-citation>)
    • <page-count> (replaced by <size>)
  • Et al. — In previous Publishing Tag Sets, the model of <etal> was defined as EMPTY, using %etal-model;. Many journals model <etal> as an EMPTY element, and it is frequently used to generate text such as “et al.” from a stylesheet. However, a few journal tag sets (Blackwell’s, for example) expect content for this element, with such text as “Associates, coworkers, and colleagues”. Therefore, to allow publishers to choose whether <etal> contains text or not, the following changes were made:
    • Deleted %etal-model; from journalpubcustom-models3.ent.
    • This left the default definition of <etal> in common3.ent as .: 
      (#PCDATA %etal-elements;)*
    • The default definition of %etal-elements; was left as all of the rendition elements such as face markup.
  • Size Versus Page Count — Now that the <size> element exists within <product>, <mixed-citation>, <element-citation>, <related-article>, and <related-object> (through %references.class;), there is no need to carry the EMPTY <page-count> element as well; so <page-count> can be reserved for metadata uses. Therefore, in EMPTYjournalpubcustom-classes3.ent" />, removed <page-count> from %references.class;.
  • Parameter Entity Modification — Implementor’s Note: In previous Tag Set versions, the parameter entity %citation-elements; was only used to create mixed-content, so by convention it started with an "|" bar. Now it may be used mixed with #PCDATA or used as an entire element-only model, so the initial “|” bar has been removed. For Publishing, removed from the “|” from the beginning of %citation-elements; in common3.ent and also added the “|” to the <mixed-citation> model.
  • Person Group — In journalpubcustom-models3.ent, added <string-name> to <person-group> (using %string-name.class;) so that the punctuation and spacing between <surname> and <given-names> could be captured.

Publishing Attribute Changes

  • Article — In journalpubcustom-models3.ent, changed the attribute @article-type in %article-atts; from a fixed set of values to CDATA. Therefore, in journalpublishing3.dtd, deleted the parameter entity article-types.
  • Article Types — In journalpublishing3.dtd, removed the parameter entity that named the article types and made the value of the @article-type on the element <article> into CDATA. 
    <!ENTITY % article-atts
"article-type 
CDATA #IMPLIED
  • Citation Types — Removed the override for %citation-atts; as it is now the same as the default, which is described below.
    The previous values for the citation-type used on <mixed-citation> and <nlm-citation> were confusing, mixing together values such as “book” and “thesis” that described the type of publication and “gov” that described the publisher. There was no way to say that something was a government-published book. Therefore, in journalpubcustom-models3.ent, the attribute citation-type was deleted from %citation-atts;, and the following attributes were added for all three types of citations (<mixed-citation>, <element-citation>, and <nlm-citation>):
    • @publication-type — The type of object (journal, book, patent) containing the cited material. While not constrained, typical values include: journal, book, patent, report, thesis, standard, website, book-part (a chapter or an article from a book where the entire book is not being cited), conf-proc (conference proceeding), working paper, list (list serve), blog, online ref (an online reference such as Wikipedia or an online dictionary), court-case, database, communc (a personal communication such as a conversation, letter, or personal email), or discussion (implies group communication, usually face to face, as opposed to an online list discussion). This is the most important of the new attributes and best practice is now to use it whenever possible, as downstream processing many require this information.
    • @publisher-type — The type of organization that publishes the cited material. While not constrained, typical values include: government (gov), standards body (std-body), personal imprint (personal), society, non-profit, ngo, etc. Unless it is important that the cited material is of a particular type, this attribute need not be used. For example, while it may be true that a cited journal was published by a commercial publisher (value “commercial”), that is rarely recorded. This attribute can preserve information about the publisher where a citation has recorded this material.
    • @publication-format — What format of the reference is being cited, for example, print, online, video, audio, etc. This attribute should be used when known for non-print sources.
    • @content-type — Provides a way to preserve or add semantic distinctions.
  • Graphic Attributes — In journalpubcustom-models3.ent,
    • Removed the attribute alternative-form-of from:
      • <graphic> (using %graphic-atts;)
      • <inline-graphic> (using %inline-graphic-atts;).
    • Also removed the attribute alt-version from <graphic> (using %graphic-atts;).
    • Also added @specific-use to <graphic> (using %graphic-atts;).
  • Orientation — In journalpubcustom-models3.ent, added a new attribute @orientation, that takes the values “portrait” and “landscape”.
    The attribute was added to %display-atts; (therefore adding it to <boxed-text>, <chem-struct-wrap>, <fig>, <fig-group>, <graphic>, <media>, <preformat>, <supplementary-material>, <table-wrap>, and <table-wrap-group>).
    @orientation was also added to <array> through default (display3.ent).
  • In journalpubcustom-models3.ent, added @specific-use to:
    • <graphic> (using %graphic-atts;)
    • <sec> (using %sec-atts;)
